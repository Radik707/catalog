## Задача 3.1 — Деплой на Vercel (~30 мин)

### Что сделать
1. Создать **приватный** репозиторий на GitHub (название: `catalog`)
2. Подключить локальный Git (`C:\catalog`) к GitHub:
   ```powershell
   cd C:\catalog
   git remote add origin https://github.com/<username>/catalog.git
   git push -u origin main
   ```
3. Зарегистрироваться / войти на [vercel.com](https://vercel.com)
4. Подключить GitHub-репозиторий `catalog` в Vercel
5. Добавить переменные окружения в настройках Vercel (Settings → Environment Variables):
   - `GOOGLE_SHEET_ID` — ID таблицы из `.env.local`
   - `GOOGLE_API_KEY` — ключ API из `.env.local`
   - Все остальные переменные из `.env.local`
6. Запустить деплой, проверить что сайт открывается по адресу `*.vercel.app`

### Ожидаемый результат
Сайт доступен по URL вида `catalog-xxx.vercel.app`, все товары загружаются, корзина работает, заказ отправляется в Telegram.

### Внимание
- `.env`, `.env.local`, `credentials.json` должны быть в `.gitignore` — проверить перед push!
- `node_modules/` и `.next/` тоже не должны попасть в Git
- После деплоя сайт будет публично доступен — задачу 3.2 делать сразу

## Задача 3.2 — Секретная ссылка UUID (~1 час)

### Что сделать
1. Сгенерировать UUID (можно через PowerShell: `[guid]::NewGuid()`)
2. Добавить переменную окружения `CATALOG_SECRET` с этим UUID:
   - Локально: в `.env.local`
   - На Vercel: в Settings → Environment Variables
3. Перестроить маршрутизацию:
   - Текущий каталог на `app/page.tsx` → перенести в `app/catalog/[secret]/page.tsx`
   - Корзина `app/cart/page.tsx` → перенести в `app/catalog/[secret]/cart/page.tsx`
   - На `app/page.tsx` — поставить заглушку ("Страница не найдена" или редирект на 404)
4. В `app/catalog/[secret]/page.tsx` добавить проверку:
   ```
   Если params.secret !== process.env.CATALOG_SECRET → redirect на 404
   Иначе → показать каталог
   ```
5. В `app/catalog/[secret]/layout.tsx` — общий макет с шапкой, передавать `secret` в CartIcon и другие компоненты для формирования ссылок
6. Все внутренние ссылки (корзина, навигация) должны включать UUID в путь

### Ожидаемый результат
- `site.vercel.app/catalog/a1b2c3d4-...` → открывается каталог
- `site.vercel.app/` → "Страница не найдена"
- `site.vercel.app/catalog/неправильный-uuid` → "Страница не найдена"
- Все внутренние ссылки работают корректно с UUID в URL

### Внимание
- UUID хранить ТОЛЬКО в переменных окружения, не в коде
- Проверку делать на стороне сервера (серверный компонент Next.js)
- После деплоя на Vercel проверить оба сценария (правильный/неправильный UUID)

## Задача 3.3 — Спецметки badges.json (~1-2 часа)

### Что сделать

**Часть A — Скрипт upload.py:**
1. Создать файл `scripts/badges.json`:
   ```json
   {
     "хит": [],
     "новинка": [],
     "акция": []
   }
   ```
   (Олег заполнит названиями товаров позже)
2. В `upload.py` добавить чтение `badges.json`
3. При записи в Google Sheet — добавить колонку `Badge` (после существующих колонок)
4. Для каждого товара проверить: если название содержит строку из `badges.json` → записать соответствующую метку ("хит", "новинка", "акция"). Поиск по вхождению подстроки, регистронезависимый
5. Если товар не найден ни в одном списке → колонка Badge пустая

**Часть B — Сайт:**
1. В `lib/types.ts` — добавить поле `badge?: 'хит' | 'новинка' | 'акция'` в тип Product
2. В `app/api/products/route.ts` — читать колонку Badge из Google Sheet
3. В `components/ProductCard.tsx` — отображать бейджик если `badge` есть:
   - **Хит** — красный ярлычок в правом верхнем углу карточки
   - **Новинка** — зелёный ярлычок
   - **Акция** — оранжевый ярлычок
   - Ярлычок: небольшой, не перекрывает название и цену, с текстом метки
4. Бейджик должен выглядеть аккуратно на мобильной версии

### Ожидаемый результат
- `python scripts/upload.py` читает `badges.json` и записывает метки в Google Sheet
- На карточках товаров из списка отображаются цветные бейджики
- Товары без меток выглядят как раньше

### Внимание
- `badges.json` должен быть в Git (это не секрет)
- Если `badges.json` отсутствует — скрипт должен работать без ошибок (просто без меток)
- Названия в `badges.json` — поиск по частичному совпадению (содержит подстроку)

## Задача 3.4 — Фото товаров через Cloudinary (~2-4 часа)

### Подготовка (Олег делает сам)
1. Зарегистрироваться на [cloudinary.com](https://cloudinary.com) (бесплатный план)
2. Получить: Cloud Name, API Key, API Secret
3. Подготовить фото: переименовать файлы понятно (например, `choco-pie.jpg`, `rollton-chicken.jpg`)

### Что сделать

**Часть A — Загрузка фото в Cloudinary:**
1. Создать скрипт `scripts/upload_images.py`:
   - Читает все jpg/png из указанной папки (аргумент командной строки или `C:\catalog\photos\`)
   - Загружает каждое фото в Cloudinary в папку `catalog/`
   - Автоматически сжимает (Cloudinary делает это сам)
   - Выводит список загруженных файлов с URL
2. Добавить переменные в `.env`:
   ```
   CLOUDINARY_CLOUD_NAME=xxx
   CLOUDINARY_API_KEY=xxx
   CLOUDINARY_API_SECRET=xxx
   ```
3. Добавить `cloudinary` в `requirements.txt`

**Часть B — Маппинг фото к товарам:**
1. Создать файл `scripts/image_map.json`:
   ```json
   {
     "Чоко Пай Орион 12шт": "choco-pie",
     "Роллтон лапша куриная": "rollton-chicken"
   }
   ```
   Ключ = название товара (или часть), значение = имя файла без расширения
2. В `upload.py` добавить чтение `image_map.json`
3. При записи в Google Sheet — добавить колонку `ImageURL`
4. Для каждого товара: если найден в `image_map.json` → записать полный URL Cloudinary. Иначе → пустая строка
5. Поиск по частичному совпадению (как в badges.json)

**Часть C — Отображение на сайте:**
1. В `lib/types.ts` — добавить поле `imageUrl?: string` в тип Product
2. В `app/api/products/route.ts` — читать колонку ImageURL
3. В `components/ProductCard.tsx`:
   - Если `imageUrl` есть → показать фото товара (сверху карточки или слева)
   - Если нет → показать иконку-заглушку (как сейчас)
   - Фото должно быть оптимизировано: использовать `next/image` с Cloudinary URL
4. Добавить домен Cloudinary в `next.config.mjs`:
   ```js
   images: {
     remotePatterns: [
       { protocol: 'https', hostname: 'res.cloudinary.com' }
     ]
   }
   ```

### Ожидаемый результат
- `python scripts/upload_images.py` загружает фото в облако
- `python scripts/upload.py` добавляет URL фото в Google Sheet
- Карточки товаров с фото — показывают фото, остальные — иконку-заглушку
- Фото корректно отображаются на мобильных устройствах

### Внимание
- Cloudinary free plan: 25 000 трансформаций/месяц, 25 ГБ хранения — более чем достаточно
- `image_map.json` в Git, ключи Cloudinary — в `.env` и `.gitignore`
- Если `image_map.json` отсутствует — скрипт работает без ошибок
- При добавлении новых фото: положить файл в папку → добавить в `image_map.json` → запустить `upload_images.py` → запустить `upload.py`
