## Подготовка (Олег делает вручную)

### Шаг 0.1 — Создать бота в Telegram
1. Открой Telegram → найди @BotFather → отправь /newbot
2. Введи имя бота (например: Каталог Олега)
3. Введи username (например: catalog_oleg_bot) — должен заканчиваться на _bot
4. Скопируй токен (длинная строка цифр и букв)

### Шаг 0.2 — Узнать свой Chat ID
1. Найди в Telegram бота @userinfobot
2. Отправь ему любое сообщение
3. Он ответит твоим Chat ID (число) — запиши его

### Шаг 0.3 — Получить Gemini API ключ
1. Открой https://ai.google.dev
2. Нажми "Get API Key" → "Create API key"
3. Скопируй ключ

### Шаг 0.4 — Создать Vercel KV
1. Открой https://vercel.com → твой проект catalog
2. Storage → Create → KV → Create
3. Скопируй KV_REST_API_URL и KV_REST_API_TOKEN

### Шаг 0.5 — Добавить переменные в .env.local
Открой C:\catalog\.env.local и добавь строки:
```
TELEGRAM_BOT_TOKEN=сюда_токен_от_BotFather
TELEGRAM_WEBHOOK_SECRET=любая_случайная_строка_например_abc123xyz
OWNER_CHAT_ID=сюда_твой_chat_id
GEMINI_API_KEY=сюда_ключ_от_google
KV_REST_API_URL=сюда_url_от_vercel
KV_REST_API_TOKEN=сюда_токен_от_vercel
```

### Шаг 0.6 — Добавить те же переменные в Vercel
1. Vercel → проект catalog → Settings → Environment Variables
2. Добавь все 6 переменных с теми же значениями

## Этап 1 — Скелет бота

### Промпт 1.1 — Установка зависимостей и структура
```
Установи npm-пакеты: grammy, @google/generative-ai, @vercel/kv

Затем создай структуру папок для Telegram-бота:
bot/
  index.ts
  handlers/
    start.ts
    catalog.ts
    cart.ts
    order.ts
  ai/
    consultant.ts
    tools.ts
    system-prompt.ts
  services/
    products.ts
    cart-store.ts
    notify.ts

Пока создай только пустые файлы с комментарием что будет в каждом.
Прочитай CLAUDE.md — там есть раздел про Telegram-бота с полным описанием архитектуры.
```

### Промпт 1.2 — Webhook endpoint
```
Создай app/api/bot/route.ts — это webhook endpoint для Telegram.

Он должен:
1. Принимать POST-запросы от Telegram
2. Проверять секрет (TELEGRAM_WEBHOOK_SECRET из env)
3. Передавать update в обработчик grammY из bot/index.ts
4. Возвращать 200 OK

Также создай bot/index.ts — инициализация бота grammY:
- Создать экземпляр Bot с токеном из TELEGRAM_BOT_TOKEN
- Экспортировать функцию handleUpdate(update) для вызова из webhook
- Подключить обработчик /start из bot/handlers/start.ts
```

### Промпт 1.3 — Команда /start
```
Создай bot/handlers/start.ts — обработчик команды /start.

При /start бот отправляет:
"👋 Привет! Я помощник по каталогу кондитерских изделий.

Могу помочь выбрать товар, подсказать цены и оформить заказ.

Выберите действие:"

И показывает inline-кнопки (InlineKeyboard из grammY):
- 📋 Каталог — callback: "catalog"
- 💬 Спросить консультанта — callback: "ask_ai"  
- 🛒 Корзина — callback: "cart"
- 📤 Отправить заказ — callback: "order"

При нажатии на "ask_ai" — бот отвечает:
"Напишите ваш вопрос, и я помогу! Например:
• Какие дешёвые конфеты есть?
• Что популярное из карамели?
• Добавь 5 кг Барбариса в корзину"
```

### Промпт 1.4 — Регистрация webhook
```
Создай скрипт scripts/setup-webhook.ts (или .js) который:
1. Берёт TELEGRAM_BOT_TOKEN из .env.local
2. Отправляет запрос на https://api.telegram.org/bot{TOKEN}/setWebhook
3. URL webhook: https://catalog-khaki.vercel.app/api/bot
4. secret_token: значение TELEGRAM_WEBHOOK_SECRET из .env.local
5. Выводит результат в консоль

Также добавь в package.json скрипт:
"setup-webhook": "npx tsx scripts/setup-webhook.ts"

После создания файла НЕ запускай его — я запущу сам после деплоя.
```

## Этап 2 — Каталог в боте

### Промпт 2.1 — Сервис товаров
```
Создай bot/services/products.ts — сервис для чтения товаров.

Он должен:
1. Читать товары из Google Sheet (переиспользуй логику из lib/sheets.ts)
2. Кэшировать результат на 15 минут (простой in-memory кэш через переменную и timestamp)
3. Экспортировать функции:
   - getProducts() — все товары
   - getCategories() — список уникальных групп (категорий)
   - getProductsByCategory(category) — товары одной категории
   - searchProducts(query) — поиск по названию (нечёткий, без учёта регистра)

Формат товара тот же что на сайте: name, price, stock, category, group, supplier, badge, imageUrl
Товары с остатком ≤ 1 не показывать (как на сайте).
```

### Промпт 2.2 — Обработчик каталога
```
Создай bot/handlers/catalog.ts — навигация по каталогу через inline-кнопки.

Callback "catalog":
1. Получить список категорий через getCategories()
2. Показать inline-кнопки — по одной на каждую категорию
3. Каждая кнопка: callback_data = "cat:{название_категории}"

Callback "cat:{категория}":
1. Получить товары этой категории через getProductsByCategory()
2. Показать список товаров (максимум 10 на страницу):
   Формат каждого товара:
   "{название}
    💰 {цена}₽ | 📦 {остаток} | {фасовка}
    [➕ В корзину]"
   
3. Если товаров больше 10:
   Кнопки пагинации: [← Назад] [Стр. 1/3] [Далее →]
   callback_data: "cat:{категория}:page:{номер}"

4. Кнопка "⬅️ К категориям" внизу — возврат к списку категорий

Кнопка "В корзину" — callback_data: "add:{product_name_encoded}:1"
(добавляет 1 единицу, количество можно будет изменить в корзине)
```

## Этап 3 — Корзина и заказы

### Промпт 3.1 — Хранилище корзины
```
Создай bot/services/cart-store.ts — корзина в Vercel KV.

Ключ: "cart:{telegram_user_id}"
Формат значения: JSON-массив [{productName, price, quantity, unit}]
TTL: 24 часа (корзина автоматически очищается через сутки)

Экспортировать функции:
- getCart(userId) → массив товаров или пустой массив
- addToCart(userId, product, quantity) → обновлённая корзина
  Если товар уже есть — увеличить количество
- removeFromCart(userId, productName) → обновлённая корзина
- updateQuantity(userId, productName, delta) → обновлённая корзина
  delta: +1 или -1, если количество = 0 — удалить товар
- clearCart(userId) → пустая корзина
- getCartTotal(userId) → {items: число, total: сумма}

Использовать @vercel/kv. Все операции обернуть в try/catch.
```

### Промпт 3.2 — Обработчик корзины
```
Создай bot/handlers/cart.ts — управление корзиной.

Callback "cart":
1. Получить корзину через getCart(userId)
2. Если пустая: "🛒 Корзина пуста. Выберите товары в каталоге!"
3. Если есть товары, показать:
   "🛒 Ваша корзина:

    1. Карамель Барбарис — 5 кг × 89₽ = 445₽
    2. Ирис Золотой ключик — 3 кг × 112₽ = 336₽
    
    💰 Итого: 781₽"

4. Для каждого товара inline-кнопки: [➖] [количество] [➕] [🗑]
5. Внизу кнопки:
   [📤 Отправить заказ]
   [🗑 Очистить корзину]
   [📋 Продолжить покупки]

Callback "add:{product}:{qty}" — добавить товар из каталога
Callback "cart_plus:{product}" — увеличить количество
Callback "cart_minus:{product}" — уменьшить количество
Callback "cart_remove:{product}" — удалить товар
Callback "cart_clear" — очистить (с подтверждением: "Очистить корзину? [Да] [Нет]")
```

### Промпт 3.3 — Обработчик заказа
```
Создай bot/handlers/order.ts — оформление и отправка заказа.

Создай bot/services/notify.ts — отправка уведомления Олегу.

Callback "order":
1. Получить корзину
2. Если пустая — "Корзина пуста, нечего отправлять"
3. Показать итог и спросить подтверждение:
   "📋 Ваш заказ:
    [список товаров с ценами]
    💰 Итого: {сумма}₽
    
    Отправить заказ?"
   Кнопки: [✅ Подтвердить] [❌ Отменить]

Callback "order_confirm":
1. Сформировать сообщение для Олега через notify.ts:
   "🛒 Новый заказ!
    
    👤 Клиент: {first_name} {last_name} (@{username})
    📱 Telegram ID: {user_id}
    
    📋 Заказ:
    1. Карамель Барбарис — 5 кг × 89₽ = 445₽
    2. Ирис Золотой ключик — 3 кг × 112₽ = 336₽
    
    💰 Итого: 781₽
    🕐 Время: {дата и время}"

2. Отправить Олегу через bot.api.sendMessage(OWNER_CHAT_ID, ...)
3. Клиенту: "✅ Заказ отправлен! Олег свяжется с вами для подтверждения."
4. Очистить корзину
```

## Этап 4 — ИИ-консультант

### Промпт 4.1 — Системный промпт
```
Создай bot/ai/system-prompt.ts — генерация системного промпта для Gemini.

Экспортировать функцию buildSystemPrompt(products) которая возвращает строку.

Системный промпт (на русском):
"Ты — опытный консультант по кондитерским изделиям. Ты помогаешь владельцам магазинов выбрать товары для закупки.

Твои знания:
- Ты знаешь все товары из каталога (данные ниже)
- Ты знаешь цены, остатки, фасовку, описания
- Ты можешь рекомендовать товары по запросу
- Ты можешь добавлять товары в корзину и оформлять заказы

Правила:
- Отвечай кратко и по делу, это деловая переписка
- Цены и остатки бери ТОЛЬКО из данных ниже, не выдумывай
- Если товара нет в данных — скажи что его нет в наличии
- Если клиент просит добавить товар — используй функцию add_to_cart
- Если клиент просит отправить заказ — используй функцию send_order
- Рекомендуй товары с хорошим остатком (больше 10 единиц)
- Если непонятно какой именно товар — уточни у клиента

Категории товаров: {список категорий}

Данные каталога:
{товары в формате: название | цена | остаток | категория | фасовка | описание}"

Товары брать из getProducts(). Формат компактный — не JSON, а простой текст через |
чтобы экономить токены. Максимум первых 200 товаров (остальные через search).
```

### Промпт 4.2 — Function calling (tools)
```
Создай bot/ai/tools.ts — определение функций для Gemini function calling.

Экспортировать массив tools в формате Gemini (functionDeclarations):

1. search_products:
   - description: "Поиск товаров по названию, категории или описанию"
   - параметр: query (string, обязательный)
   - Реализация: вызывает searchProducts(query) из products.ts

2. add_to_cart:
   - description: "Добавить товар в корзину клиента"
   - параметры: product_name (string), quantity (number, default 1)
   - Реализация: ищет товар через searchProducts → addToCart()
   - Если найдено несколько — вернуть список для уточнения

3. remove_from_cart:
   - description: "Удалить товар из корзины"
   - параметр: product_name (string)
   - Реализация: removeFromCart()

4. show_cart:
   - description: "Показать текущую корзину клиента"
   - Реализация: getCart() → форматировать

5. send_order:
   - description: "Оформить и отправить заказ владельцу"
   - Реализация: getCart() → notify() → clearCart()

Каждый tool возвращает текстовый результат для Gemini.
```

### Промпт 4.3 — Обработчик ИИ-сообщений
```
Создай bot/ai/consultant.ts — основной ИИ-обработчик.

Экспортировать функцию handleAIMessage(ctx) для grammY.

Логика:
1. Получить текст сообщения от пользователя
2. Получить текущие товары (из кэша) через getProducts()
3. Собрать системный промпт через buildSystemPrompt(products)
4. Отправить в Gemini Flash:
   - model: "gemini-2.0-flash"
   - systemInstruction: системный промпт
   - tools: из tools.ts
   - contents: сообщение пользователя
5. Если Gemini вернул function call:
   - Выполнить функцию (передав userId из ctx)
   - Вернуть результат функции в Gemini
   - Получить финальный текстовый ответ
6. Отправить ответ клиенту в Telegram
7. При ошибке: "Извините, произошла ошибка. Попробуйте позже или выберите товары через кнопку «Каталог»."

Также обнови bot/index.ts:
- Все текстовые сообщения (не команды) → handleAIMessage(ctx)
- Добавить typing indicator: ctx.replyWithChatAction("typing") перед запросом к Gemini

Используй @google/generative-ai пакет.
API ключ из process.env.GEMINI_API_KEY.
```
