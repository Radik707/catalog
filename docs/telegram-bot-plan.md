# Telegram-бот «Каталог-консультант» — Полный план

> Дата: 3 апреля 2026
> Проект: расширение каталога catalog-khaki.vercel.app

---

## Архитектура

```
Клиент (Telegram) 
    ↓ пишет сообщение
Telegram API
    ↓ webhook
Vercel API Route: /api/bot/route.ts
    ↓
Обработчик бота (grammY)
    ├── Команды (/start, /cart, /order)
    ├── Кнопки (каталог, категории, товары)
    └── Свободный текст → Gemini Flash AI
            ├── Вопрос о товарах → ответ с рекомендациями
            ├── "Добавь 3 кг Адели" → добавляет в корзину
            └── "Отправь заказ" → формирует и шлёт Олегу
    
Данные:
    Google Sheets API (те же товары что на сайте)
    Vercel KV (корзина каждого клиента)

Заказ:
    Бот → Telegram Bot API → сообщение Олегу в личку
```

### Почему эта архитектура

| Решение | Почему |
|---------|--------|
| grammY (не python-telegram-bot) | TypeScript, живёт в том же Next.js проекте, не нужен отдельный сервер |
| Webhook (не polling) | Vercel = serverless, polling невозможен. Webhook — запрос пришёл → обработали → ответили |
| Gemini Flash (не Claude) | Бесплатно 15 req/min + 1M tokens/day. Для 10-50 B2B клиентов хватит с запасом |
| Vercel KV (корзина) | Бесплатно 30K запросов/мес. Корзина хранится между сообщениями |
| Тот же Vercel проект | Не нужен новый хостинг, тот же код доступа к Google Sheets |

## Этапы разработки

### Этап 1 — Скелет бота (1-2 дня)

**Цель:** бот отвечает на /start и показывает меню с кнопками.

**Подготовка (Олег делает вручную):**
1. Открыть Telegram → найти @BotFather → /newbot
2. Придумать имя (например «Каталог Олега») и username (например catalog_oleg_bot)
3. Скопировать токен бота
4. Добавить в C:\catalog\.env.local: `TELEGRAM_BOT_TOKEN=полученный_токен`

**Задачи для Claude Code:**
- Установить зависимости: grammy, @vercel/kv
- Создать app/api/bot/route.ts — webhook endpoint
- Создать bot/index.ts — инициализация grammY
- Создать bot/handlers/start.ts — /start с кнопками:
  - 📋 Каталог
  - 💬 Спросить консультанта
  - 🛒 Корзина
  - 📤 Отправить заказ
- Настроить webhook (скрипт или команда для регистрации URL)

**Результат:** бот отвечает на /start красивым меню.

---

### Этап 2 — Каталог в боте (2-3 дня)

**Цель:** клиент нажимает «Каталог» → видит категории → товары с ценами и остатками.

**Задачи для Claude Code:**
- Создать bot/services/products.ts:
  - Читает товары из Google Sheet (переиспользовать логику из lib/sheets.ts)
  - Кэширует на 15 минут (чтобы не долбить API на каждое сообщение)
  - Возвращает товары, группированные по категориям
- Создать bot/handlers/catalog.ts:
  - Кнопка «Каталог» → список 11 категорий (inline-кнопки)
  - Клик на категорию → список товаров (название, цена, остаток, фасовка)
  - Пагинация если товаров больше 10 (кнопки «←» «→»)
  - Кнопка «В корзину» у каждого товара

**Результат:** клиент может листать каталог и видеть актуальные цены/остатки.

---

### Этап 3 — Корзина и заказы (2-3 дня)

**Цель:** клиент добавляет товары → видит корзину → отправляет заказ Олегу.

**Подготовка:**
- Создать Vercel KV хранилище (Олег делает в панели Vercel)
- Добавить KV_REST_API_URL и KV_REST_API_TOKEN в .env.local

**Задачи для Claude Code:**
- Создать bot/services/cart-store.ts:
  - Ключ: `cart:{telegram_user_id}`
  - Операции: добавить, убрать, очистить, получить
  - Формат: массив `[{productName, price, quantity, unit}]`
- Создать bot/handlers/cart.ts:
  - Кнопка «Корзина» → список товаров с количеством и суммой
  - Кнопки +/− для каждого товара
  - Кнопка «Очистить корзину»
  - Итого внизу
- Создать bot/handlers/order.ts:
  - Кнопка «Отправить заказ» → подтверждение
  - Формирует сообщение: имя клиента + список товаров + сумма
  - Отправляет Олегу в Telegram (через bot.api.sendMessage)
  - Клиенту: «✅ Заказ отправлен! Олег свяжется с вами»
  - Очищает корзину
- Создать bot/services/notify.ts:
  - Форматирование заказа в читаемый текст
  - Отправка Олегу (OWNER_CHAT_ID в .env.local)

**Результат:** полный цикл заказа через кнопки.

---

### Этап 4 — ИИ-консультант (3-5 дней)

**Цель:** клиент пишет свободный текст → ИИ отвечает и может управлять корзиной.

**Подготовка:**
- Зарегистрироваться на ai.google.dev → получить API-ключ Gemini
- Добавить GEMINI_API_KEY в .env.local

**Задачи для Claude Code:**
- Создать bot/ai/system-prompt.ts:
  - Системный промпт: «Ты — консультант по кондитерским изделиям...»
  - Включает: описание категорий, правила рекомендаций
  - Динамически подставляет текущие товары (из кэша)
- Создать bot/ai/tools.ts — function calling для Gemini:
  - `search_products(query)` — поиск товаров по названию/описанию
  - `add_to_cart(product_name, quantity)` — добавить в корзину
  - `remove_from_cart(product_name)` — убрать из корзины
  - `show_cart()` — показать корзину
  - `send_order()` — отправить заказ
- Создать bot/ai/consultant.ts:
  - Принимает сообщение клиента
  - Отправляет в Gemini Flash с системным промптом + tools
  - Gemini решает: ответить текстом или вызвать функцию
  - Выполняет функцию → возвращает результат Gemini → Gemini формирует ответ клиенту
  - Финальный ответ отправляется клиенту в Telegram

**Примеры разговоров после этого этапа:**

```
Клиент: "Какие дешёвые конфеты есть?"
Бот: [ИИ ищет товары с низкой ценой]
     "Вот недорогие варианты в наличии:
      1. Карамель «Барбарис» — 89₽/кг (остаток 45 кг)
      2. Ирис «Золотой ключик» — 112₽/кг (остаток 30 кг)
      3. Карамель «Дюшес» — 95₽/кг (остаток 22 кг)
      Хотите что-то добавить в корзину?"

Клиент: "Добавь барбарис 5 кг и ирис 3 кг"
Бот: [ИИ вызывает add_to_cart дважды]
     "✅ Добавлено:
      • Карамель «Барбарис» 5 кг — 445₽
      • Ирис «Золотой ключик» 3 кг — 336₽
      Итого в корзине: 781₽"

Клиент: "Отправь заказ"
Бот: [ИИ вызывает send_order]
     "📤 Заказ отправлен! Олег свяжется с вами для подтверждения."
```

**Результат:** полноценный ИИ-консультант с управлением корзиной через свободный текст.

## Когда MVP готов?

MVP (этапы 1-4) считается готовым, когда:
- ✅ Бот отвечает на /start с меню
- ✅ Можно листать каталог по категориям
- ✅ Можно добавлять товары в корзину кнопками
- ✅ ИИ отвечает на вопросы о товарах
- ✅ ИИ добавляет товары в корзину по текстовой команде
- ✅ Заказ приходит Олегу в Telegram
- ✅ Данные актуальные (из того же Google Sheet что и сайт)

## Риски и решения

| Риск | Решение |
|------|---------|
| Gemini бесплатный лимит кончится | Переключить на Claude Haiku ($5 баланс) — меняется 1 файл |
| Vercel KV лимит (30K/мес) | Для 50 клиентов это ~600 запросов/день — хватит. Если нет → Upstash |
| ИИ неправильно находит товар | Нечёткий поиск (fuzzy match) + подтверждение: «Вы имели в виду...?» |
| Webhook не срабатывает | Логирование + fallback: клиент видит «Попробуйте позже» |
| Медленный ответ (>10 сек) | Vercel hobby: 60 сек лимит. Для Gemini Flash ответ обычно 2-4 сек |
